# Antigravity Agent System Rules (system.md)

모든 Antigravity 에이전트는 역할과 실행 환경에 관계없이 아래의 전역 행동 제약 조건을 무조건 준수해야 합니다.

## 1. 프로젝트 및 환경 보존 규칙
- **임의의 라이브러리 및 도구 추가 금지**: `config.json`에 선언된 패키지 매니저(`npm`, `uv`) 이외의 임의 도구(예: yarn, pip 등)를 사용해 패키지를 설치하거나 빌드 환경을 수정하는 행위를 금지합니다.
- **오케스트레이션 제어 파일 보호**: `.antigravity/` 디렉토리 하위의 설정 파일(`config.json`), 실행 파일(`runner.py`), 훅 스크립트(`hooks/` 하위), 스킬 규칙(`skills/` 하위)을 임의로 삭제하거나 오케스트레이션 동의 없이 임의 수정하지 않습니다.

## 2. 상태 무결성 및 태스크 보존 규칙
- **독점적 진행 원칙**: 한 번에 오직 하나의 태스크만 `"진행중"` 상태여야 합니다. 이전 태스크의 리뷰가 완전히 승인되어 `"완료"`로 갱신되기 전까지 다음 태스크로 건너뛰거나 동시에 여러 태스크의 파일들을 변경하지 않습니다.
- **포맷 임의 변형 금지**: `.agents/checklist.yml` 및 `progress.md`를 업데이트할 때, 기존 파일의 필드 구조나 데이터 타입을 훼손하지 않습니다.

## 3. TDD 파이프라인 준수 규칙
- **테스트 생략 금지**: `tester.md`가 실패하는 테스트 코드를 생성하기 전에 `coder.md`가 먼저 실제 코드를 개발하는 편법을 절대 금지합니다.
- **테스트 결과의 객체성 신뢰**: `reviewer.md`가 코드 리뷰 판정을 내릴 때, 에이전트 자체의 추론에 의존하여 통과 여부를 임의 판정하지 않고, `config.json`에 정의된 실제 테스트 러너 실행 명령어의 exit code 결과(`0`이면 성공, 이외 실패)를 기반으로만 검증 결과를 기록합니다.

## 4. 로깅 투명성 규칙
- **에러 로그 상세화**: 빌드 에러나 테스트 실패가 발생했을 때 이를 숨기지 않습니다. 터미널 표준 에러 출력(stderr) 전체를 `log.md`에 가감 없이 정확히 기입하여 디버깅 피드백 컨텍스트로 제공합니다.
