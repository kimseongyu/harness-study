# Git Commit Formatter Rule

에이전트가 기능을 성공적으로 구현하거나 문서를 작성하고 Git 커밋을 진행할 때, 커밋 메시지의 가독성과 일관성을 유지하기 위해 적용하는 서식 규칙입니다.

## 1. 커밋 제목 포맷 (Commit Subject)
제목은 50자 이내로 요약하며, `Conventional Commits` 기반으로 작성합니다.

```
[type]([scope]): [subject] [requirement_id]
```

- **Type**:
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `docs`: 문서 추가 및 수정
  - `refactor`: 코드 리팩토링 (기능 변화 없음)
  - `test`: 테스트 코드 추가 및 수정
- **Scope**: 변경이 일어난 영역 중 해당하는 항목 하나를 기입합니다 (`frontend`, `ai-server`, `harness`, `config`).
- **Subject**: 한글 명사형 종결로 끝나는 간결한 설명으로 작성합니다.
- **Requirement ID**: 관련된 요구사항 식별자가 있을 경우 제목 끝에 명시합니다.

*예시*:
- `feat(frontend): 상단 네비게이션 탭 구현 REQ-01-F01`

## 2. 본문 작성 규칙 (Commit Body)
- 상세한 변경 사항이 필요한 경우 제목과 한 줄 띄우고 작성합니다.
- 변경된 핵심 파일 목록 및 리팩토링의 핵심 근거를 개조식(bullet points)으로 명시합니다.
