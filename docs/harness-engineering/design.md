# Material Design 기반 UI/UX 가이드라인

이 문서는 모바일 화면에서 구동되는 Chrome 공룡 게임의 모든 네이티브 및 웹뷰 UI 컴포넌트가 일관되고 유려한 외관을 갖출 수 있도록 **Google의 Material Design 3 (M3)** 스펙을 이식 및 설계한 가이드라인입니다.

---

## 1. 디자인 시스템 원칙

모바일 T-Rex 러너 게임의 사용자 경험을 극대화하기 위해 M3의 핵심 개념 세 가지를 적용합니다:
1. **Color (색상 대비 및 접근성):** 어두운 환경과 밝은 야외 환경 모두에서 공룡과 선인장 장애물이 뚜렷하게 분별될 수 있도록 높은 대비 비율을 제공합니다.
2. **Shape (형태와 둥근 모서리):** 카드, 버튼, 팝업 모달 등에 부드럽고 친근한 코너 반경(Rounded Corner)을 일관되게 부여합니다.
3. **Ergonomics (터치 타겟과 조작성):** 긴박한 게임 상황에서 터치 오동작이 일어나지 않도록 터치 타겟(Touch Target) 크기를 엄격히 제어합니다.

---

## 2. 컬러 시스템 (Material 3 Color Palette)

모든 컴포넌트(Native UI 및 Canvas UI)는 아래 테마 토큰 색상을 준수하여 렌더링해야 합니다.

### 2.1 라이트 및 다크 모드 통합 팔레트

| 컬러 토큰 | HSL 값 / HEX 코드 | 용도 및 컴포넌트 적용 |
| :--- | :--- | :--- |
| **Primary** | `hsl(210, 100%, 40%)` / `#005faf` | 주요 점수 강조, 액티브 점프 트리거 영역 |
| **Secondary** | `hsl(215, 15%, 50%)` / `#6c7784` | 게임 일시정지(Pause) 뱃지, 부차적 제어 버튼 |
| **Background** | `hsl(0, 0%, 98%)` / `#fafafa` | 게임 전체 배경화면 및 최상위 컨테이너 |
| **Surface** | `hsl(0, 0%, 100%)` / `#ffffff` | 설정 다이얼로그 모달, 카드 레이아웃 배경 |
| **On Surface** | `hsl(0, 0%, 10%)` / `#1a1a1a` | 기본 텍스트, 아이콘, 장애물(선인장/익룡) 드로잉 색상 |
| **Outline** | `hsl(215, 10%, 80%)` / `#cbd2d9` | 컴포넌트의 테두리, 구분선 및 충돌 임시 경계선 |

## 3. 타이포그래피 (Typography Scale)

텍스트 크기 계층은 모바일 장치의 가독성을 저해하지 않도록 설계합니다. (기본 폰트: `System Default Sans-Serif` 또는 `Roboto`)

- **Display Large (999+ 최고 기록 및 게임오버 알림):** `32sp`, Bold, 자간 `-0.5px`, 줄간격 `40px`
- **Headline Medium (스코어보드 실시간 점수):** `24sp`, Medium, 자간 `0px`, 줄간격 `32px`
- **Title Medium (모달 팝업 및 헤더 타이틀):** `16sp`, Semi-Bold, 자간 `0.15px`, 줄간격 `24px`
- **Body Large (설정 라벨 및 긴 텍스트):** `14sp`, Regular, 자간 `0.25px`, 줄간격 `20px`
- **Label Medium (버튼 텍스트 및 메타 정보):** `12sp`, Medium, 자간 `0.5px`, 줄간격 `16px`

## 4. 컴포넌트 세부 디자인 가이드

Material Design 3 스펙에 기반한 개별 UI 컴포넌트 구현 명세입니다.

### 4.1 터치 반응 버튼 (M3 Button with Ripple)
- **크기 (Touch Target):** 모든 모바일 대화형 버튼은 최소 가로/세로 **`48dp x 48dp`** 이상의 터치 유효 영역을 확보해야 합니다.
- **둥글기 (Corner Radius):** 완전한 둥근 알약 형태(`border-radius: 9999px`) 또는 `12dp` 값을 사용합니다.
- **마이크로 인터랙션:** 터치 시작 시 크기가 95%로 수축(`scale(0.95)`)되고 떼면 원래 크기로 탄성 복원되는 마이크로 인터랙션을 제공합니다.

### 4.2 오버레이 바텀 시트 및 모달 (M3 Bottom Sheet)
- **형태:** 상단 코너에 `28dp` 크기의 둥근 곡률(Top Left / Top Right Rounded)을 적용하여 화면 하단에서 슬라이드 업 형태로 노출됩니다.
- **그림자 (Elevation):** 컴포넌트에 입체감을 높이기 위해 `Elevation Level 3` (Material Design Shadow 6%)을 적용합니다.

### 4.3 스코어보드 배지 (M3 Badge/Chip)
- **레이아웃:** 화면 우측 상단 배치. `12dp` 곡률을 지닌 작은 알약형 컨테이너(Primary HSL 기반 투명도 10% 배경) 내부에 검은색 숫자로 모노스페이스(Monospace) 스타일 폰트를 사용하여 실시간 점수를 깔끔하게 표기합니다.

## 5. UI/UX 일관성 검증 하네스 요건

이 디자인 규격서에 부합하는지 테스트 하네스에서 검증할 때 다음과 같은 비주얼 체크 시나리오를 따릅니다.
1. **터치 타겟 크기 검사:** 네이티브 버튼 컴포넌트의 클릭 영역이 CSS/스타일상 `48dp` 미만으로 구현되었는지 자동 크기 확인.
2. **콘트라스트 비율 검증:** 텍스트 색상과 배경 색상의 대비 비율이 WebAIM 기준 **`4.5:1` 이상**을 만족하는지 컬러 토큰 값을 조회하여 테스트 패널에서 검토.
3. **가로/세로 스케일 복원력:** 노치 디자인(Safe Area Insets) 적용 디바이스에서 스코어보드나 제어 패널이 상단 카메라 영역이나 하단 소프트키 바에 겹치지 않고 최소 `16dp` 패딩 안전 마진을 지키는지 확인.
