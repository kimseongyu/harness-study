# Chrome 공룡 게임 기능 및 입력 요구사항 명세서 (Native)

이 문서는 모바일 화면에서 구동되는 Chrome 공룡 게임(T-Rex Runner)의 핵심 물리 엔진 법칙, 장애물 특성, 점수 계산 공식 및 모바일 터치 입력 요구사항을 정의합니다. 하네스 테스트를 설계할 때 이 명세서의 물리 값과 작동 로직을 기준으로 검증해야 합니다.

---

## 1. 게임 개요 및 핵심 루프

Chrome 공룡 게임은 횡스크롤 러너 게임으로, 플레이어가 조종하는 공룡(T-Rex)이 우측에서 좌측으로 다가오는 장애물을 점프하거나 고개를 숙여 피하며 생존하는 게임입니다.

- **프레임 레이트:** 모바일 기기 기준 target 60FPS (프레임 드롭 발생 시 물리 속도 스케일링 필요)
- **핵심 루프:** 
  1. 사용자 입력(Native Touch/Gesture) 감지 및 T-Rex 상태 변화
  2. 물리 연산 (T-Rex 위치 계산, 장애물 이동)
  3. 충돌 감지 (AABB Boundary Collision Checking)
  4. 점수 업데이트 및 난이도 상승 (속도 증가)
  5. 렌더링 (React Native State 업데이트를 통한 네이티브 View 리렌더링)

---

## 2. 물리 엔진 상세 규격

하네스 엔지니어링을 통한 검증이 가능하도록 물리 수치 상수를 명확히 정의합니다.

### 2.1 중력 및 점프 물리
- **초기 속도 (Initial Velocity, $V_0$):** $V_0 = 12 \text{ px/frame}$ (점프 시작 시 상향 속도)
- **중력 가속도 (Gravity, $g$):** $g = 0.6 \text{ px/frame}^2$ (점프 중 공룡을 아래로 당기는 힘)
- **공룡 Y 좌표 계산식:** 
  $$Y_{t} = Y_{t-1} + V_{t}$$
  $$V_{t} = V_{t-1} - g$$
  *(단, Y좌표는 바닥면 $Y_{\text{ground}}$보다 커질 수 없음. 모바일 화면 크기에 따라 스케일 조정이 가능함)*
- **공중 머무름 시간 (Hang Time):** 최고점 도달 후 지면 착지까지 약 `40 frames` (약 0.66초)

### 2.2 장애물 이동 속도 (Difficulty Scaling)
- **시작 속도 (Initial Speed):** $6.0 \text{ px/frame}$
- **최대 속도 (Max Speed):** $13.0 \text{ px/frame}$
- **속도 가속도 (Acceleration Rate):** 프레임당 $+0.001 \text{ px/frame}$ (또는 100점 달성 시마다 속도 가속 적용)

## 3. 모바일 터치 및 입력 컨트롤 명세

데스크톱의 Spacebar/Up Arrow, Down Arrow 입력을 모바일의 터치 제스처로 완벽히 매핑해야 합니다. React Native 컴포넌트의 `onTouchStart`, `onTouchMove`, `onTouchEnd`를 활용해 직접 처리합니다.

| 입력 수단 | 모바일 제스처 | 매핑 동작 | 설명 |
| :--- | :--- | :--- | :--- |
| **Space / Up** | **싱글 탭 (Single Tap)** | 점프 (Jump) | 공룡이 Y축 방향으로 솟구쳐 오름 (공중에 있는 상태에서 탭 시 2단 점프 미지원) |
| **Down** | **아래로 스와이프 (Swipe Down)** 및 유지 | 숙이기 (Duck) | 공룡이 납작하게 엎드려 익룡(Pterodactyl) 등 공중 장애물을 회피함. 터치를 떼면 즉시 일어섬 |

### 터치 반응성 요구사항 (Latency Spec)
- 터치 입력이 React Native View 단에서 감지되어 공룡 액션으로 전환되는 데 걸리는 지연 시간(Input Latency)은 **30ms 이하**여야 합니다. (WebView Bridge가 배제되어 입력 반응성이 크게 향상됨)
- 스와이프 인식 임계치(Threshold): 스와이프 제스처 판정을 위한 최소 드래그 거리 `30px`, 판정 시간 `150ms` 이내.

## 4. 장애물 (Obstacles) 종류 및 충돌 판정

### 4.1 장애물 규격

1. **선인장 (Cactus - Small & Large):**
   - 지면 장애물.
   - 단일(Single), 이중(Double), 삼중(Triple) 배치로 스폰.
   - 가로 넓이: `15px ~ 30px`, 세로 높이: `24px ~ 40px`. (스케일 인자에 따라 네이티브 화면에 맞춰 변환)
2. **익룡 (Pterodactyl - Bird):**
   - 공중 장애물. 3가지 다른 높이 수준에서 생성됨 (낮음-점프로만 피함, 중간-점프 혹은 숙임, 높음-숙이기로 피함).
   - 날개짓 애니메이션 프레임 주기: `160ms` (약 10프레임).

### 4.2 충돌 판정 (Collision Box)
충돌은 AABB(Axis-Aligned Bounding Box) 기준을 따르되, 공룡의 스프라이트 외곽선 공백으로 인한 오판정을 막기 위해 바운딩 박스를 실제 이미지의 약 **85% 크기(Inner Collision Box)**로 수축하여 적용합니다.

```text
+-----------------------+
|  T-Rex (Original BB)  |
|   +-----------------+ |
|   | Inner Box (85%) | |
|   |  [충돌 감지 영역]  | |
|   +-----------------+ |
+-----------------------+
```

## 5. 점수 및 저장 시스템 (Score System)

- **점수 획득 공식:** 생존 프레임 수 기준 `5 frames = 1 point` 획득.
- **체크포인트 피드백:** 100점 단위 돌파 시 React Native 단에서 점수 갱신 이벤트 리스너를 실행하고, 디바이스 햅틱(진동) 피드백 제공 가능.
- **최고 점수 (Highscore):** 
  - 게임 오버 시 React Native Native Realm의 `AsyncStorage`로 최고 점수 저장 및 영구 동기화.

## 6. 하네스 테스트를 위한 체크포인트

이 규격에 근거하여 작성될 하네스는 다음을 실시간 시뮬레이션할 수 있어야 합니다.
1. **중력 테스트:** 공룡의 점프 도달 시간 및 낙하 물리 상수가 $g=0.6$에 맞게 움직이는지 시계열 데이터 추적.
2. **충돌 시뮬레이션:** 선인장과 공룡의 Bounding Box가 $1\text{px}$ 단위로 겹칠 때 정확하게 `GameOver` 상태로 전이되는지 검사.
3. **네이티브 터치 지연 감지:** 모바일 터치 이벤트 발생 시점과 게임 루프 동작 시점 간 타임스탬프 편차 분석.
