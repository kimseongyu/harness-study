# 통합 테스트 시나리오 및 검증 케이스

이 문서는 React Native 및 WebView 기반 Chrome 공룡 게임의 기능성, 반응성, 성능 및 예외 동작을 철저히 검증하기 위한 상세 테스트 케이스 시나리오집입니다.

---

## 1. 기능적 테스트 케이스 (Functional Test Cases)

### 1.1 초기화 및 화면 진입
- **케이스 ID:** `TC-FUN-001`
- **검증 목적:** 앱 실행 시 게임 자산이 정상 로드되고 대기 화면이 표시되는지 확인.
- **수행 절차:**
  1. 기기의 네트워크 연결을 차단(오프라인 모드).
  2. 앱을 빌드 후 최초 실행.
  3. 로딩 스피너가 사라진 뒤 캔버스 중앙에 대기 중인 공룡 T-Rex가 보이는지 검사.
- **기대 결과:** `GAME_READY` 브릿지 메시지가 송신되고, 화면 확대 없이 모바일 스케일에 맞춘 최적 크기로 렌더링될 것.

### 1.2 점프 및 고점프 동작 물리 검증
- **케이스 ID:** `TC-FUN-002`
- **검증 목적:** 터치 입력의 길이 및 횟수에 따른 점프 궤적이 정확히 실행되는지 검사.
- **수행 절차:**
  1. 게임 화면을 가볍게 탭(Single Tap) -> T-Rex의 최고 도달 Y축 높이와 착지 프레임 수 기록.
  2. 게임 화면을 길게 탭 후 터치 유지(Tap & Hold) -> 최고 도달 Y축 높이 비교.
- **기대 결과:** 일반 탭 시 약 `40 frames` 동안 체공해야 하며, 길게 탭 했을 때 도달 높이가 일반 탭보다 15% 이상 높게 날아야 함.

### 1.3 충돌 및 게임 오버 상태 전이
- **케이스 ID:** `TC-FUN-003`
- **검증 목적:** 장애물과 충돌 시 즉시 게임 루프가 중단되고 네이티브로 데이터가 전달되는지 확인.
- **수행 절차:**
  1. 게임을 시작하여 공룡을 움직임.
  2. 다가오는 선인장(Cactus)과 점프 타이밍을 맞추지 않고 직접 충돌을 유도함.
- **기대 결과:** 
  - 충돌 즉시 게임 Canvas가 정지하며 공룡이 눈을 찌푸린 형태(`Collision Sprite`)로 변경됨.
  - Native 단에 `GAME_OVER` JSON 데이터가 전송되며, 점수가 AsyncStorage 내 최고 점수(Highscore)보다 높을 경우 신기록이 즉시 갱신되어야 함.

## 2. 성능 및 반응성 테스트 케이스 (Non-Functional Test Cases)

### 2.1 터치 반응 레이턴시 테스트
- **케이스 ID:** `TC-PERF-001`
- **검증 목적:** 사용자가 화면을 탭한 직후 실제 공룡이 점프를 시작하는 순간의 시간 지연을 측정하여 최적화 수준 판단.
- **수행 절차:**
  - 에뮬레이터 검사 도구(Inspect 콘솔)에 `onTouchStart` 타임스탬프와 공룡 물리 상태 변화(`jump_started = true`) 시점의 타임스탬프 차이를 기록하는 성능 코드를 삽입하고 구동.
- **기대 결과:** 평균 지연 시간이 **80ms 이하**로 유지되어 플레이에 이질감이 없어야 함.

### 2.2 프레임 드롭(FPS) 및 하드웨어 가속 검증
- **케이스 ID:** `TC-PERF-002`
- **검증 목적:** 고득점(속도가 최대치 $13.0\text{ px/frame}$에 수렴할 때) 상황에서 끊김 없이 60FPS가 보장되는지 확인.
- **수행 절차:**
  1. 하네스 패널을 통해 강제로 게임 속도를 최대로 조절.
  2. 화면에 선인장 및 익룡 장애물을 다량 스폰함.
  3. Native 개발자 도구의 Perf Monitor에서 GPU 및 UI 프레임 레이트 감시.
- **기대 결과:** 장애물이 급증하는 연출 중에도 FPS가 **55FPS 이하로 떨어지지 않고** 일정하게 유지될 것.

## 3. 예외 및 극한 상황 테스트 케이스 (Edge Cases)

### 3.1 백그라운드 전환 시 일시 정지 (Lifecycle Test)
- **케이스 ID:** `TC-EXC-001`
- **검증 목적:** 홈 화면 이동이나 전화 수신 등으로 앱이 백그라운드로 진입했다가 복귀(Foreground) 시 게임 강제 오버 방지.
- **수행 절차:**
  1. 플레이 중(점수 300점 돌파 시점) 모바일 홈 버튼을 눌러 앱을 백그라운드로 전환.
  2. 10초 대기 후 다시 앱 아이콘을 탭해 복귀시킴.
- **기대 결과:** 복귀 시 게임은 일시정지(`PAUSE`) 상태여야 하며, 사용자가 화면을 다시 터치해야만 진행되도록 설계되어 충돌사를 예방해야 함.

### 3.2 디바이스 화면 회전 (Rotation Locking)
- **케이스 ID:** `TC-EXC-002`
- **검증 목적:** 화면 회전 시 캔버스 스케일이 일그러지는 버그 예방.
- **수행 절차:**
  - 기기의 화면 회전 잠금을 해제하고 가로 모드(Landscape)로 강제 회전시킴.
- **기대 결과:** 앱 레이아웃이 세로 모드(Portrait)로 고정되어 화면 왜곡이 없거나, 가로 모드로 변환되더라도 캔버스의 비율이 유실되지 않고 정중앙 정렬을 유지해야 함.
