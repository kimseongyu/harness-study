# REQ-01 반려동물 AI 이미지 생성 및 보관 서비스

> **[안내]** 본 문서는 반려동물 사진을 생성하고 확인하는 웹 서비스의 요구사항 명세서입니다.

## 1. 개요 및 목표
- **작성 일자**: 2026-06-29
- **상태**: 기획 중 (Draft)
- **목적**: 고양이, 강아지, 수달, 토끼 등 다양한 반려동물의 이미지를 사용자가 선택한 키워드 스타일에 맞춰 생성 및 다운로드하고, 생성 이력을 저장하여 확인할 수 있는 웹 서비스를 구축합니다.
- **대상 사용자**: 귀여운 동물 사진을 보거나 수집하는 것을 좋아하는 일반 사용자.

## 2. 사용자 스토리 (User Story)
- **시나리오**:
  > "나는 **일반 사용자**로서, **원하는 종류의 동물 이미지를 내가 고른 분위기로 다양하게 얻기** 위해 **동물과 키워드 버튼을 눌러 이미지를 생성**하고 싶다. 이를 통해 **원하는 사진을 다운로드하거나 이전에 생성한 히스토리를 한눈에 볼 수 있는** 가치를 얻는다."
- **주요 흐름 (Happy Path)**:
  1. 사용자가 웹 서비스에 접속하면 상단 네비게이션 바에서 원하는 동물(예: 고양이)을 선택합니다.
  2. 화면 하단의 10개 키워드 버튼 중 하나(예: "귀여운")를 클릭합니다.
  3. 화면 중앙 뷰어 영역에 "귀여운 고양이" 이미지가 생성되어 렌더링됩니다.
  4. 사용자는 다운로드 버튼을 눌러 이미지를 로컬로 저장하거나, 새로고침을 통해 새로운 이미지를 다시 생성합니다.
  5. 화면 하단의 히스토리 갤러리 영역에서 방금 생성된 이미지와 함께 기존에 생성된 고양이 이미지 목록을 확인합니다.

## 3. 기능 요구사항 (Functional Requirements)
이 요구사항을 구현하기 위해 필요한 하위 세부 기능 목록입니다.

- [ ] **REQ-01-F01 상단 동물 네비게이션 바**:
  - 화면 상단 고정 영역에 고양이, 강아지, 수달, 토끼 메뉴 배치.
  - 각 메뉴 클릭 시 해당하는 동물의 이미지 생성 화면으로 라우팅/전환.
- [ ] **REQ-01-F02 이미지 뷰어**:
  - 화면 정중앙에 생성된 이미지를 보여주는 전용 창 구성.
  - 이미지 생성 전에는 대기 상태(Placeholder) 또는 안내 텍스트 노출.
  - 생성 중에는 로딩 인디케이터/애니메이션 노출.
- [ ] **REQ-01-F03 스타일 키워드 버튼**:
  - 뷰어 하단에 10개의 스타일 버튼 배치.
  - 키워드 목록: "귀여운", "하찮은", "용감한", "우아한", "졸린", "행복한", "신비로운", "뚱한", "장난꾸러기", "터프한".
  - 버튼 클릭 시 현재 선택된 동물 종류와 해당 키워드를 바탕으로 이미지 생성 API 요청.
- [ ] **REQ-01-F04 이미지 다운로드 및 새로고침**:
  - 뷰어 영역 근처에 '다운로드' 및 '새로고침' 버튼 구현.
  - 다운로드 클릭 시 현재 활성화된 이미지를 로컬 PC에 저장 (`[동물]_[키워드]_[타임스탬프].jpg` 포맷).
  - 새로고침 클릭 시 직전에 선택한 키워드로 이미지를 재요청하여 생성.
- [ ] **REQ-01-F05 이미지 파일 분류 및 저장 (Backend/Harness)**:
  - 생성된 이미지를 서버 로컬의 `images/[동물명]/` 디렉토리에 자동으로 분류하여 저장.
- [ ] **REQ-01-F06 이전 생성 이미지 갤러리**:
  - 생성 화면 아래에 현재 선택된 동물의 이전 생성 파일 목록을 로드하여 그리드(Grid) 레이아웃으로 표시.
  - 새로운 이미지가 생성/저장될 때마다 갤러리가 자동으로 갱신됨.

## 4. 비기능 요구사항 (Non-functional Requirements)
- **성능**: 이미지 생성 API는 비동기로 처리되며, 생성 중 브라우저가 멈추지 않아야 함.
- **디자인 시스템**: Sleek Dark Mode와 부드러운 Glassmorphism 디자인 적용, 마우스 호버 시 미세한 스케일 업 애니메이션 적용.
- **반응형**: 데스크톱 기준 중앙 집중형 구조를 지향하되 모바일 세로 모드에서도 버튼 10개가 2열 배치 등으로 깨지지 않고 노출되어야 함.

## 5. 인수 조건 (Acceptance Criteria)
1. **인수 조건 1**: 상단 '수달' 탭을 클릭하고 하단 '하찮은' 버튼을 누르면, 수달 캐릭터/실사 풍의 '하찮은' 분위기 이미지가 중앙 뷰어에 나타나야 한다.
2. **인수 조건 2**: 다운로드 버튼을 누르면 브라우저의 기본 다운로드 동작을 통해 이미지가 저장되어야 한다.
3. **인수 조건 3**: 이미지가 성공적으로 저장되면, 하단 갤러리의 가장 앞(혹은 뒤)에 방금 생성된 수달 이미지가 즉시 추가되어 확인 가능해야 한다.
